API Endpoints — Full Detailed Examples
=====================================

This file contains every DTO, exact request/response JSON examples, and suggested mock filenames you can drop into a frontend mock folder.

Structure
- Section per resource (Auth, Cars, Rentals, Payments)
- For each endpoint: HTTP method + path, purpose, request JSON (exact), response JSON (exact), suggested mock filename
- DTO definitions are included as JSON schema-like examples and as the Java record definition for precision

---
AUTH
---
POST /auth/login
Purpose: manager login -> returns JWT
Java DTO (LoginRequest):
public record LoginRequest(String email, String password) {}

Exact Request JSON (file: mock/auth/login_request.json):
{
  "email": "manager@amc.local",
  "password": "password"
}

Exact Response JSON (file: mock/auth/login_response.json):
{
  "token": "eyJ...<jwt>..."
}

Notes: HTTP 401 on bad credentials.

PAYMENTS
---
Java DTO (PaymentRequestDto):
public record PaymentRequestDto(
  Long rentalId,
  com.example.amc.model.Payment.DeliveryMethod deliveryMethod,
  String deliveryCity
) {}

Java DTO (PaymentResponseDto):
public record PaymentResponseDto(
  Long id,
  Long rentalId,
  java.math.BigDecimal amountToPay,
  java.math.BigDecimal deliveryFee,
  com.example.amc.model.Payment.DeliveryMethod deliveryMethod,
  String deliveryCity,
  com.example.amc.model.Payment.Status status,
  com.example.amc.model.Payment.Type type,
  java.time.LocalDateTime paidAt
) {}

Notes: Payments endpoints require MANAGER auth for creating and marking paid. Listing and viewing payments are public.

1) POST /payments?rentalId={id}&deliveryMethod={AGENCY|DELIVERY}&deliveryCity={city} (MANAGER)
- deliveryMethod default: AGENCY
- If deliveryMethod=DELIVERY and deliveryCity=Agadir then deliveryFee = 30.00 added to amount

Example 1 (agency)
Request (query params): rentalId=8, deliveryMethod=AGENCY

USAGE NOTES FOR FRONTEND
- Manager-only actions (create/update/delete car, create payment, mark payment, return rental) must include the `Authorization: Bearer <jwt>` header.
- Public actions (browse cars, create rental, view rentals/payments) require no auth header.
- Use sample files above to seed frontend mock server or to feed into fixture data.
